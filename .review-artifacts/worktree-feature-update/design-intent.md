# 설계의도

## 작업 개요

AOS Dashboard 및 백엔드의 기술 부채 정리와 일관성 개선 작업.
크게 4가지 독립적인 개선을 하나의 feature branch에서 수행한다:

1. Dashboard 전역 `font-mono` 클래스 제거 (UI 일관성)
2. ACE Framework 완전 폐기 (사용되지 않는 거버넌스 모델 제거)
3. 프로젝트 등록 메커니즘 강화 (e2e 테스트 잔여물 방지)
4. Dashboard 통계 정확도 개선 (세션 프로젝트 API 활용)

## 핵심 설계 결정

### 결정 1: font-mono 클래스를 UI 컴포넌트에서 제거 (코드 컨텍스트 예외)

- **선택지**: (A) 코드/터미널 컨텍스트만 유지, 나머지 제거 vs (B) 전체 일괄 제거
- **선택**: A — 코드/터미널 컨텍스트는 예외로 유지
- **이유**: 일반 UI 요소(라벨, ID 표시, 경로 텍스트 등)에서는 시스템 기본 폰트로 통일하되, **코드 diff, 터미널 로그, YAML/JSON 에디터, `<pre>`/`<code>` 블록** 등 모노스페이스가 필수적인 컨텍스트에서는 `font-mono`를 유지해야 정렬과 가독성이 보장됨.
- **적용 범위**: 63개 파일에서 일반 UI의 font-mono를 제거. 단, DiffViewer, OutputLog, YamlEditor 등 코드 컨텍스트에서의 제거는 **리뷰 시 지적 대상** (과도 적용).
- **리뷰 관점**: 코드 컨텍스트에서 font-mono가 제거된 부분은 수정이 필요한 결함으로 취급.

### 결정 2: ACE Framework 흔적 없이 완전 제거

- **선택지**: (A) deprecated 표시만 추가 vs (B) 코드/타입/UI 완전 삭제
- **선택**: B — 완전 삭제
- **이유**: ACE(Autonomous Cognitive Entity) Framework는 4-Pillar 거버넌스 모델로 설계되었으나, 실제 운영에서 활용되지 않고 코드 복잡도만 증가시켰음. 백엔드 모델(`ace_capabilities` 필드), 프론트 타입/UI, 스킬 파일, 워크스페이스 템플릿 에이전트 설정을 모두 삭제하여 깨끗한 상태로 정리.
- **영향 범위**: `project_config.py`, `agent_manager.py`, `project_config_monitor.py`, `AgentsTab.tsx`, `projectConfigs.ts` 스토어, 테스트 모킹 데이터, `.claude/skills/ace-framework/` 전체.

### 결정 3: projects/ 디렉토리에서 e2e 잔여물 구분 로직 강화

- **선택지**: (A) 디렉토리 + symlink 모두 등록(기존) vs (B) symlink만 등록 vs (C) e2e 잔여물만 필터링
- **선택**: C 방향으로 수정 필요 (현재 구현은 B)
- **배경**: e2e 테스트가 `projects/` 디렉토리에 일반 디렉토리를 생성한 뒤 정리하지 않으면, 해당 디렉토리가 실제 프로젝트로 등록되는 버그 발생.
- **현재 구현의 문제**: symlink만 허용하면 `git clone`으로 직접 프로젝트를 생성하거나 디렉토리를 복사하는 정상 워크플로우까지 차단됨. 이는 의도와 다름.
- **올바른 방향**: e2e 잔여물을 식별하는 별도 로직(예: 마커 파일, 명명 규칙, 생성 시간 기반 필터)을 도입하거나, e2e 테스트 자체가 잔여물을 확실히 정리하도록 보장.
- **보완**: `ProjectCleanupService`는 잔여 일반 디렉토리도 `shutil.rmtree`로 정리할 수 있도록 보완하여, 이미 등록된 비정상 프로젝트의 삭제를 지원.
- **리뷰 관점**: symlink 전용 필터링은 **과도한 제약으로 리뷰 시 지적 대상**. git clone/디렉토리 복사 경로도 지원해야 함.

### 결정 4: Dashboard 세션 프로젝트 통계를 전용 API로 교체

- **선택지**: (A) 세션 목록에서 추출(기존) vs (B) 전용 프로젝트 목록 API 사용
- **선택**: B — 전용 API (`fetchSessionProjects`)
- **이유**: 기존 방식은 세션의 `project_name` 필드에서 unique set을 추출했는데, 프로젝트명이 `-`이거나 누락된 세션이 있어 부정확. 전용 API는 실제 등록된 프로젝트 목록을 반환하므로 정확도 향상.
- **폴백**: `allSessionProjects`가 비어있으면 기존 세션 기반 추출 방식으로 폴백하여 하위 호환성 유지.
- **에러 핸들링**: API 호출 실패 시 빈 배열이 반환되면 폴백이 동작하지 않는 문제가 있음. **에러 상태를 별도로 체크**하여 API 실패 시에도 폴백이 확실히 동작하도록 해야 함. (리뷰 시 지적 대상)

## 의도적으로 제외한 것

| 항목 | 제외 이유 |
|------|-----------|
| CSS custom font 설정 | font-mono 제거만 수행, 별도 웹폰트 도입은 이번 범위 밖 |
| ACE 대체 거버넌스 모델 | ACE 제거 후 새 모델 도입은 별도 작업으로 분리 |
| e2e 테스트 자체 수정 | 잔여물 방지는 등록 메커니즘에서 처리, 테스트 코드 수정은 별도 |
| 다른 Zustand 스토어 리팩토링 | projectConfigs 스토어의 ACE 타입만 제거, 전체 리팩토링은 범위 밖 |
| `.temp/` 디렉토리 정리 | 임시 데이터이므로 커밋 대상에서 제외 |

## 주의사항

1. **font-mono 과도 제거 (결함)**: DiffViewer, OutputLog, YamlEditor 등 코드/터미널 컨텍스트에서 font-mono가 제거된 것은 의도와 다름. 코드 정렬과 가독성을 위해 이 컨텍스트들은 font-mono를 복원해야 함. 리뷰에서 해당 파일들을 식별하여 수정 요청 필요.
2. **ACE 필드 제거와 API 호환성**: 기존 API 응답에 `ace_capabilities` 필드가 포함되었으므로, 프론트엔드 이전 버전과의 호환성 주의. (현재 프론트/백이 같은 브랜치에서 변경되므로 문제없음)
3. **symlink 전용 등록이 과도함 (결함)**: git clone이나 디렉토리 복사로 프로젝트를 등록하는 정상 워크플로우가 차단됨. e2e 잔여물만 필터링하는 방식으로 수정 필요. 리뷰에서 대안 제시 필요.
4. **DashboardPage 에러 핸들링 미흡 (결함)**: `fetchSessionProjects` API 실패 시 에러 상태를 별도 체크하지 않아, 빈 배열과 실패를 구분할 수 없음. 에러 상태 기반 폴백 로직 추가 필요.
